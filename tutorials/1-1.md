# Tutorial 1.1 - Build Tetris with Pygame

In this tutorial you’ll build a playable **Tetris** game step by step. You’ll see why **classes** are useful for organizing the game, and you’ll practice **problem-solving** for:

- **Storing** the board and pieces
- **Rotating** pieces using offset lists
- **Preventing** rotation into walls or other blocks (try → check → rollback)
- **Clearing** full rows (we’ll show the bug first, then fix it)

Complete the [lessons]({{ site.baseurl }}/lessons/1-basics/1-types-of-data/) first if you haven’t already (variables, conditionals, loops, lists, functions). You’ll need Pygame installed and basic knowledge of **classes**, **lists**, **dictionaries**, and **loops**.

## When you're done

You'll have a game you can run from your own script. Play it, show it to a friend, and try the ideas below to make it feel more like *yours*.

**Make it yours (optional):**
- Set the window title to your name: `pygame.display.set_caption("Mia's Tetris")`
- Change the block colours in `render()` or when you draw the board
- Add a simple sound when a row clears: `pygame.mixer` and a short beep or use a free sound from the web
- Flash the row (draw it white for one frame) right before it disappears - that "juice" makes the game feel more satisfying

The reward is running something you built. Every time you move a piece or clear a row, that's your code doing it.
